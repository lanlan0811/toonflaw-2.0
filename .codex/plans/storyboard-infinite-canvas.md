# 分镜表“无限画布编辑生成”优化计划

## 概要

- 在“分镜表列表”的操作栏新增“编辑生成”，打开全屏无限画布。
- 前端功能在 `data/web` 实现；允许同步修改后端源码、测试及构建产物。
- 复用现有图片上传、图片生成、工作流保存和分镜更新接口。

## 核心实现

### 分镜列表与数据接口

- 在 `data/web/secondary-dev-patch.js` 的每条分镜操作中增加“编辑生成”，传入当前分镜、项目、剧本及关联资产。
- 修改 `src/routes/storyboardImport/list.ts`，在列表响应中补充可空的 `flowId`，用于跨浏览器、跨设备恢复画布。
- 后端重新构建 `data/serve/app.js`；不新增数据库字段或迁移。
- 画布开启时暂停列表轮询，关闭后恢复并刷新列表，避免编辑期间数据被重绘。

### 无限画布模块

- 新增独立的画布 JS/CSS，由 `data/web/index.html` 加载，并通过内部桥接接口供分镜列表调用。
- 支持画布平移、25%–200% 缩放、节点拖拽、适配视图、自动布局、撤销/重做。
- 节点类型包括：
  - 图片节点：关联角色、场景、道具或用户新增的本地图片。
  - 生成节点：提示词、引用关系、模型、比例、质量、生成结果和最终结果标记。
- 连接限定为“图片或生成结果 → 生成节点”，禁止重复连接、自连接和循环依赖。
- 默认按“角色→场景→道具”创建当前分镜的关联资产节点，并连接到一个默认生成节点。
- 默认生成节点继承当前分镜提示词和缩略图，并默认标记为最终结果。
- 允许新增、删除节点和连接；关闭存在未保存修改的画布时弹出确认。

### 资产覆盖与提示词引用

- 图片节点支持上传 JPEG/PNG；上传成功后标记为画布覆盖图。
- 后续生成只使用覆盖图 URL，不再传入原资产图；不调用任何资产修改接口，因此资产库中的原图不变。
- 提供“恢复原资产”操作；只有用户主动恢复后才重新使用原图。
- 提示词编辑器采用结构化引用：
  - 输入 `@` 展示当前连入生成节点的图片。
  - 插入 `@图片N`，悬停显示缩略图和资产名称。
  - 引用内部绑定稳定的节点 ID，节点排序或换图不会使引用错位。
  - 删除连接后对应引用显示失效状态，并阻止生成，直到删除或重新连接。
- 调用现有 `generateFlowImage` 时，将结构化内容编译为普通提示词，并按引用顺序传递图片 URL。
- 每个生成节点独立选择模型、比例和质量；优先使用项目默认配置，模型列表加载失败时保留项目默认模型作为兜底。
- 同一生成节点同一时间只允许一个生成任务；失败信息显示在节点内，不自动替换旧结果。

### 保存、恢复和兼容

- 用户必须将一个具有有效图片的生成节点设为“最终结果”；生成中、引用失效或最终图为空时禁用全局“保存”。
- 保存顺序固定为：
  1. 使用 `saveImageFlow` 或 `updateImageFlow` 保存完整节点、连接、视口和元数据。
  2. 使用 `editStoryboardInfo` 保存最终节点编译后的提示词，保持原 `videoDesc`。
  3. 使用 `updateStoryboardUrl` 写入最终图片 URL 和 `flowId`。
  4. 刷新分镜列表，使对应行缩略图立即更新，然后关闭画布。
- 任一步失败时保留画布和生成结果，展示错误并允许幂等重试，不提前更新列表中的可见缩略图。
- 再次进入时根据列表返回的 `flowId` 恢复完整画布，不依赖浏览器本地存储。
- 对旧版图片工作流进行兼容转换：
  - 尝试把旧图片节点匹配到当前关联资产。
  - 普通上传节点保留为自定义图片。
  - 旧提示词转换为结构化文本。
  - 优先将与当前分镜缩略图匹配的节点识别为最终结果，否则选取最近的有效生成节点。
- 恢复时同步当前资产关联：新增关联资产自动补入；已解除关联的系统节点移除；用户主动删除过的关联节点不自动复活。

## 接口与持久化约定

- `/api/storyboardImport/list` 每条记录新增 `flowId: number | null`，其余字段保持兼容。
- 前端内部暴露 `window.ToonflowStoryboardCanvas.open(config)`，负责画布生命周期；它不是对外公共 API。
- 工作流继续存入现有 `o_imageFlow`：
  - 节点保存图片、原资产信息、覆盖状态、提示词结构、生成参数、最终结果标记。
  - 边保存源节点、目标节点和引用顺序。
  - 画布元数据保存版本号、视口、关联资产快照及用户删除记录。
- 不修改角色、道具、场景资产本身，不把上传覆盖图写回资产库。

## 验证方案

- 自动检查：
  - 为分镜列表响应补充测试，覆盖有效 `flowId`、空 `flowId` 及项目/剧本隔离。
  - 对新增和修改的浏览器 JS 执行语法检查。
  - 运行 `yarn lint`、`yarn test:storyboard-import` 和相关测试。
  - 执行项目构建并检查生成差异，只保留本功能需要的构建产物。
- 手工验收：
  - 从任意分镜正确进入画布，默认节点、图片和提示词与该行一致。
  - 验证拖拽、缩放、连线、删除、自动布局、撤销/重做。
  - 覆盖角色、场景、道具图片后，确认生成请求不再包含原图，资产库原图未变化。
  - 验证 `@图片N` 插入、悬停预览、换图稳定性和失效引用拦截。
  - 验证生成成功、失败重试、多生成节点及最终结果切换。
  - 保存后对应行缩略图和提示词更新；重新进入及清空浏览器存储后仍能恢复完整画布。
  - 验证旧工作流兼容、接口失败保留现场，以及“编辑关联”“强制重生”“删除”等原功能不受影响。

## 实施约束

- 前端文件仅在 `data/web` 中修改或新增；后端源码、测试和必要构建产物可同步修改。
- 开始修改前确认当前为本地 `master` 并检查工作区状态。
- 保留用户现有未提交内容，不覆盖、不删除、不纳入本功能。
- 不创建新分支，不提交或推送，除非用户后续明确要求。
